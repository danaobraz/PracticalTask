# Опис проєкту

Цей репозиторій містить два незалежних завдання, присвячених тестуванню програмних модулів та побудові базової системи моніторингу.
У першому завданні реалізовано модуль з арифметичними операціями та набір тестів для перевірки його коректності.
Друге завдання охоплює створення інфраструктури для збору, обробки та візуалізації метрик роботи бази даних та допоміжного веб-сервісу.

# 1. Опис Завдання №1: Тестування Арифметичних Операцій

Створено модуль для перевірки коректності реалізації базових математичних дій.  
У межах роботи використано кілька типів тестів, що забезпечують розширену перевірку логіки та обробки виняткових ситуацій.

## ■ Мета Роботи

- Розробити окремий модуль з арифметичними функціями.
- Реалізувати підхід до тестування, що охоплює:
  - стандартні модульні тести;
  - тести на коректність обробки виключень;
  - параметризовані тести для багаторазової перевірки однієї функції з різними параметрами.

## ■ Інструменти

- Python3
- Модуль `unittest`
- Файл тестів: **test_calculator.py**

---

# 2. Опис Завдання №2: Система Моніторингу Бази Даних

Проєкт присвячений створенню середовища, яке дозволяє генерувати навантаження на базу даних, збирати системні метрики, зберігати їх у time‑series сховищі та візуалізувати за допомогою Grafana.

## ■ Основні Компоненти

- Генератор операцій над БД (Python)
- PostgreSQL у Docker-контейнері
- Сервер авторизації (Flask + SQLite)
- Prometheus для збору метрик
- Grafana для відображення інформації

## ■ Архітектурна Структура

1. Генератор взаємодії з БД: виконує додавання, оновлення та вибірку даних.
2. База даних PostgreSQL: розгортається за допомогою Docker.
3. Сервер авторизації: реалізовано REST API для реєстрації та автентифікації.
4. Prometheus: здійснює періодичне опитування метрик сервісів.
5. Grafana: налаштовується для побудови графіків на основі даних Prometheus.

---

# ▶ Інструкції щодо Запуску

## 1. Тестування арифметичних функцій

Необхідний встановлений Python3.

```
cd task1
python -m unittest -v test_calculator.py
```

## 2. Запуск системи моніторингу

Потрібен Docker Desktop.

```
cd task2
docker-compose up --build -d
docker-compose down
```
